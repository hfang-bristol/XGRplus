Section 10 ENRICHMENT ANALYSER (DOMAINS)

10.1 Input Interface

Input

  • Step 1 and Step 2: User-input domain list, together with the domain type specified. Make sure that the input domain identifiers (Step 2) are consistent with the domain level (Step 1); otherwise, the error messages will be prompted

  • Step 3: List of ontologies supported.

  • More Controls: More parameters to fine-tune steps described above.

Output

  • Example Output includes interactive tables nd and high-quality figures for enrichment results. A summary of input data and the runtime (computed on the server side) is also returned for the reference.
